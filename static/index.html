for engineers · under the hood

Ten layers between Enter and the first token

That's where the name comes from — strata. No LangChain, no LlamaIndex: every stage below is hand-rolled, and each card names the exact file that implements it. Watch a question flow through all ten:

Reciprocal Rank Fusion

Dense and keyword result lists merged by rank — a hit on both engines always wins.

Cross-encoder re-rank

A precision model re-scores every candidate against the exact question.

MMR · λ = 0.7

Relevance balanced against novelty — six varied chunks, never six paraphrases.

Grounded-refusal gate

An evidence threshold that structurally blocks hallucination before generation.

SSE streaming

Tokens render word-by-word with live Markdown, tables and SVG charts.

Shadow DOM widget

Zero-dependency embed; host CSS can't leak in, widget can't leak out.

Vision OCR routing

Pages are classified digital / scanned / rich; GLM-OCR (0.9B) reads scans, tables & figures, Tesseract as fallback.

768-dim local embeddings

nomic-embed-text on your own GPU — no embedding API bills.

the stack behind it

Five parts, each doing one job well

Boring, proven technology — arranged carefully. FastAPI, PostgreSQL and Ollama, wired together with zero frameworks in between.

Chat widget JavaScript · Shadow DOM
A tiny, dependency-free widget any site can load. Isolated so the host page can't break it.
API server Python · FastAPI
Receives questions, checks keys, streams answers back live over SSE.
Retrieval engine hand-rolled · no frameworks
The ten-layer pipeline above — searching, fusing, re-ranking and filtering the evidence.
Storage & search PostgreSQL
One database holds the documents, their vectors and the keyword index — per project, fully isolated.
AI models Ollama · local
Understanding and writing both happen on your own hardware. No API bills, no data sharing.
integrate

Put it on your own site

One script tag for the widget, or an OpenAI-style streaming endpoint if you'd rather call the API directly.

Embed the widget

Paste a single script tag before the closing body tag of any website and the chat bubble appears on that page. Everything is white-labelled through attributes on the tag — the assistant's name, welcome message, logo and suggested questions — so no code changes are ever needed. The widget renders inside a Shadow DOM, meaning your site's styles can't break it and it can't interfere with your site.

Call the API directly

Prefer your own frontend? Every project exposes its own chat endpoint, secured by an API key sent as a bearer token. You post the conversation and the answer streams back word-by-word over Server-Sent Events — the same mechanism the widget uses. The ready-made script tag, endpoint URL and API key for each project are all available in the Admin panel under Embed.
